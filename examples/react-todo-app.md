## 1. 鸟瞰全局 (Project Map)

项目名称: React Todo App
做什么: 一个支持增删改查、筛选、本地存储的待办事项应用
技术栈: React 18 + TypeScript + Zustand + Tailwind CSS + Vite
核心思路: 用 Zustand 管理全局状态，组件只负责渲染和用户交互，数据持久化到 localStorage。

目录骨架:
```
src/
├── components/       ← UI 组件层
│   ├── TodoItem.tsx      单条待办的展示和编辑
│   ├── TodoList.tsx      待办列表容器
│   ├── TodoInput.tsx     新增待办的输入框
│   └── FilterBar.tsx     筛选栏 (All/Active/Completed)
├── stores/           ← 状态管理
│   └── todoStore.ts      Zustand store，所有待办的状态和操作
├── types/            ← 类型定义
│   └── todo.ts            Todo 接口和 Filter 类型
├── utils/            ← 工具函数
│   └── storage.ts         localStorage 读写封装
├── App.tsx           ← 根组件
└── main.tsx          ← 入口文件
```

## 2. 技术栈速成 (Tech Stack Primer)

技术: React 18
是什么: 用于构建用户界面的 JavaScript 库
为什么用它: 组件化架构让 UI 可拆分、可复用
核心概念: 每个组件是一个函数，接收 props，返回 JSX。状态变化时自动重新渲染。
深入学习: https://react.dev/learn

技术: TypeScript
是什么: JavaScript 的超集，添加了类型系统
为什么用它: 在编码阶段发现 bug，IDE 自动补全更准确
核心概念: 接口 (interface) 定义数据"长什么样"，编译器帮你检查
深入学习: https://www.typescriptlang.org/docs/

技术: Zustand
是什么: 轻量级 React 状态管理库 (仅 1KB)
为什么用它: 比 Redux 简单 10 倍，不需要 Provider/Action/Reducer 那一套
核心概念: store 就是一个自定义 hook，组件里直接 `const todos = useTodoStore(s => s.todos)` 就能用
深入学习: https://github.com/pmndrs/zustand

技术: Tailwind CSS
是什么: 原子化 CSS 框架，用 class 名直接写样式
为什么用它: 不用在 CSS 文件和组件文件之间来回跳，开发速度快
核心概念: class 名就是样式，比如 `p-4` = padding: 1rem
深入学习: https://tailwindcss.com/docs

## 3. 架构走读 (Architecture Walkthrough)

当用户点击"添加"按钮时，数据流是这样的:

1. [入口] 用户在 TodoInput 组件输入文字并按 Enter
   → 调用 handleAdd() 函数

2. [状态更新] handleAdd 调用 `useTodoStore.getState().addTodo(text)`
   → Zustand store 里执行: 创建新 Todo 对象，追加到数组，写入 localStorage
   → store 状态变更，所有订阅了 todos 的组件自动重新渲染

3. [UI 更新] TodoList 收到新的 todos 数组
   → React 渲染新的 TodoItem 组件

4. [用户看到] 页面上出现新的待办事项

```
用户输入 → 组件事件 → Store 更新 → 自动渲染 → UI 更新
    ↑                                              ↓
    └──── 用户看到新 Todo 出现在列表中 ─────────────┘
```

## 4. 关键模式解读 (Key Patterns)

### 模式 1: Zustand Store 的自动持久化

在哪: `src/stores/todoStore.ts:15-20`

是什么: 每次 store 变化时自动同步到 localStorage

为什么这么写: 用户刷新页面后数据不丢失，不需要后端

不这么写会怎样: 每次刷新页面待办事项全部消失

```typescript
// 创建 store 时用 persist 中间件自动处理 localStorage
export const useTodoStore = create<TodoState>()(
  persist(
    (set) => ({
      todos: [],
      addTodo: (text) => set((state) => ({
        todos: [...state.todos, { id: Date.now(), text, completed: false }]
      })),
      // ... 其他操作
    }),
    { name: 'todo-storage' }  // localStorage key
  )
);
```

### 模式 2: 受控组件 (Controlled Component)

在哪: `src/components/TodoInput.tsx:8-15`

是什么: 输入框的值由 React state 控制，而不是 DOM

为什么这么写: React 能准确知道输入框的当前值，方便做验证和提交

不这么写会怎样: 需要用 ref 去读 DOM 的值，代码更乱，且难以做实时验证

```typescript
// value 由 state 控制，onChange 更新 state
<input
  value={text}                    // React 控制显示什么
  onChange={(e) => setText(e.target.value)}  // 用户输入时更新 state
  onKeyDown={(e) => e.key === 'Enter' && handleAdd()}
/>
```

## 5. 动手实验 (Try It Yourself)

### 实验 1: 修改待办文字的颜色
难度: ⭐ (入门)

步骤:
1. 打开 `src/components/TodoItem.tsx`
2. 找到 `<span>` 元素
3. 在 className 里加上 `text-blue-500`
4. 保存文件，浏览器自动刷新
5. 观察: 所有待办文字变成蓝色

学到了什么: Tailwind 的 class 名就是样式，`text-blue-500` = 蓝色文字

### 实验 2: 添加"已完成数量"统计
难度: ⭐⭐ (进阶)

步骤:
1. 打开 `src/components/FilterBar.tsx`
2. 在 return 之前加: `const completedCount = todos.filter(t => t.completed).length`
3. 在 JSX 里加: `<span>已完成: {completedCount}</span>`
4. 保存，观察筛选栏多了一个计数

学到了什么: 组件可以从 store 读数据，派生计算其他值，React 会自动更新

## 6. 扩展指引 (Extension Points)

如果要添加"拖拽排序"功能:
- 需要改: `TodoList.tsx` (加拖拽事件), `todoStore.ts` (加 reorder 方法)
- 参考: react-beautiful-dnd 或 @dnd-kit/core
- 注意: 拖拽库需要给每个 item 一个稳定的 key (用 id 而不是 index)

如果要添加"后端同步"功能:
- 需要改: `todoStore.ts` (把 localStorage 换成 API 调用), 新增 `api/todo.ts`
- 参考: Zustand 的 async actions 模式
- 注意: 要处理网络错误和加载状态
